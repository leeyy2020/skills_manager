# Skills Manager 使用说明

这个项目提供了一个统一入口，用来管理 Codex 和 OpenCode 的 skills。

## 文件位置

- 顶层命令入口：[`skillsctl`](/home/ubuntu/work/skillsmanager/skillsctl)
- skill 定义：[`skills/skills-manager/SKILL.md`](/home/ubuntu/work/skillsmanager/skills/skills-manager/SKILL.md)
- 核心脚本：[`skills/skills-manager/scripts/manage_skills.py`](/home/ubuntu/work/skillsmanager/skills/skills-manager/scripts/manage_skills.py)
- 配置样例：[`skills-manager.toml.example`](/home/ubuntu/work/skillsmanager/skills-manager.toml.example)

## 快速开始

先给顶层入口执行权限：

```bash
chmod +x ./skillsctl
```

查看本机已经安装的 skills：

```bash
./skillsctl list --app all
```

查看当前实际加载了哪个配置文件：

```bash
./skillsctl show-config
```

## 配置文件

支持以下加载顺序：

1. `--config /path/to/config.toml`
2. 环境变量 `SKILLS_MANAGER_CONFIG`
3. 当前目录 `./skills-manager.toml`
4. 用户目录 `~/.config/skills-manager/config.toml`

建议先复制样例文件：

```bash
cp ./skills-manager.toml.example ./skills-manager.toml
```

一个常见配置如下：

```toml
[git]
repo = "https://github.example.com/platform/skills.git"
repo_subdir = "skills"
ref = "main"

[paths]
codex_dir = "~/.codex/skills"
opencode_dir = "~/.config/opencode/skills"

[defaults]
app = "codex"
```

配置后，很多命令就不需要重复传 `--repo`、`--repo-subdir`、`--ref`。

## 常用命令

查看已安装 skills：

```bash
./skillsctl list --app all
```

从本地目录安装一个 skill 到 Codex：

```bash
./skillsctl install-local --app codex --source /absolute/path/to/my-skill
```

卸载一个已安装 skill：

```bash
./skillsctl uninstall --app codex --name my-skill
```

查看企业版 GitHub 仓库里有哪些 skills：

```bash
./skillsctl catalog-git
```

从企业版 GitHub 安装指定 skill：

```bash
./skillsctl install-git --skill release-helper --skill jira-triage
```

从企业版 GitHub 同步整个 skills 目录到 OpenCode：

```bash
./skillsctl sync-git --app opencode --force
```

## GitHub Enterprise 认证

支持三种方式：

1. 命令行参数：`--token` 或 `--username` + `--password`
2. 环境变量：`GHE_TOKEN`
3. 环境变量：`GHE_USERNAME` + `GHE_PASSWORD`

推荐使用环境变量，不要把密钥直接写进配置文件。

例如：

```bash
export GHE_TOKEN='ghp_xxx'
./skillsctl catalog-git
./skillsctl install-git --skill skills-manager
```

脚本会通过临时 `GIT_ASKPASS` 自动把账号密码或 token 提供给 git，避免交互式输入。

## OpenCode 路径说明

OpenCode 默认会按顺序探测以下目录：

1. `~/.config/opencode/skills`
2. `~/.opencode/skills`
3. `~/.local/share/opencode/skills`

如果都不存在，则默认使用 `~/.config/opencode/skills`。也可以用环境变量 `OPENCODE_SKILLS_DIR` 覆盖。

## 建议用法

- 第一次使用先运行 `./skillsctl show-config`
- 拉远程仓库前先运行 `./skillsctl catalog-git`
- 覆盖已存在 skill 时再加 `--force`
- 敏感认证信息尽量放到环境变量，不要直接落到仓库文件
